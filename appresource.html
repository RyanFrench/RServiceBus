To Start With


*Environment variables are used for configuration
*Allow external resources for message handlers to be configured through rservicebus
*Enables easier movement of apps through custom environments,
                    eg dev, staging, prod.

Layout

                    RSB_<name>=<type>://[<username>:<password>@]<host>:<port/[<specific name>]

name: Name of the variable in the Message Handler, to which this resource will be assigned.

type: Type resource to be created, listed below

username:password: Optional, only used when relevant to the resource

host:port: where to find the resource

specific name: only used where appropriate, eg database name

                    RSB_Redis=redis://localhost:6379/

redis

Connect to a Redis server

RSB_Redis=redis://localhost:6379/

mysql

Connect to a Redis mysql

RSB_Mydb=mysql://localhost/

fluiddbmysql

Connect to a Redis mysql

RSB_Mydb=fluiddbmysql://localhost/

fluiddbmysql2

Connect to a Redis mysql

RSB_Mydb=fluiddbmysql2://localhost/

fluiddbpgsql

Connect to a Redis mysql

RSB_Mydb=fluiddbpgsql://localhost/

dir

Connect to a directory

RSB_OutputDirectory=dir:///path/to/output/

file

Connect to a file

RSB_InputCsvFile=file:///input/csv/file.csv
