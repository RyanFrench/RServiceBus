To Start With


*Environment variables are used for configuration
*Allow rservicebus to observe events and generate msgs from them

Layout

                    RSBOB_<name>=<type>://[<username>:<password>@]<host>:<port/[<specific name>]

name: Name of the msg to be generated, and therefore, which message handler it will be sent.

type: Type of observer to be created, listed below

username:password: Optional, only used when relevant to the resource

host:port: where to find the thing to be observered

specific name: only used where appropriate, eg path

                    RSBOB_Indigo=csvdir:///path/to/dir/

dir

Monitor a directory for files, and send a msg with the content

RSBOB_Indigo=dir:///path/to/dir/

csvdir

Monitor a directory for csv files, and send a msg with the content parsed as csv

RSBOB_Indigo=csvdir:///path/to/dir/

csvperlinedir

Monitor a directory for csv files, and send a msg per line

RSBOB_Indigo=csvperlinedir:///path/to/dir/

xmldir

Monitor a directory for xml files, and send a msg with the content parsed as Xml

RSBOB_Indigo=xmldir:///path/to/dir/
